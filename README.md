## 📝 Project Overview
Developed in **2023**, this project served as my foundational exploration into mobile development. The primary objective was to digitalize a physical motorcycle spare parts catalog into a portable, real-time digital system.

The application was designed to empower field teams and warehouse staff by providing instant access to product data, item verification, and inventory input workflows directly from their mobile devices.

---

## 🛠️ Technical Stack & Focus
*   **Framework:** Flutter
*   **Language:** Dart
*   **Core Focus:** 
    *   **Architecture:** Clean navigation workflows and modular UI structure.
    *   **State Management:** Handling local UI states and interactive components.
    *   **UI/UX:** Implementing grid-based catalogs, interactive modals, and professional dark-themed aesthetics.

---

## 🚀 Key Features

*   **📦 Warehouse Catalog:** A responsive grid-based view featuring spare part images, codes, and pricing tiers.
*   **🔍 Item Detail Insights:** Interactive modals displaying technical data such as *Product Codes*, *Units*, and *Cross-references*.
*   **📥 Inventory Input:** A dedicated form module designed for efficient digital registry of new warehouse stock.
*   **🗺️ Multi-Module Navigation:** A structured drawer-based navigation system for seamless transitions between operational modules.
